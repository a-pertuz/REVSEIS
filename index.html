Overview

This documentation hub provides installation guides and essential information for three specialized seismic data processing tools. Each tool addresses a specific need in seismic data workflows:

SEGYRecover

Python Windows

A tool for digitizing scanned seismic sections and converting them to standard SEGY format.

VELRecover

Python Windows

A tool for interpolating 2D seismic velocity data from sparse velocity analysis found in seismic sections.

InSeis

Python Windows WSL

A graphical interface for creating and running Seismic Unix workflows on Windows through WSL.

Common Prerequisites

Installing Python (Required for all tools)

Download Python from python.org
During installation, check "Add Python to PATH"
Click "Install Now" and wait for installation to complete
Verify installation by opening Command Prompt and typing: python --version

Note: Python 3.12 or higher is recommended for all tools.

SEGYRecover

▼

A Python tool for digitizing scanned seismic sections and converting them to standard SEGY format.

GitHub Repository Features Detailed Tutorial

Installation

Open Command Prompt (search for "cmd" in Windows search) and enter:

pip install segyrecover

Alternatively, install directly from GitHub:

pip install git+https://github.com/a-pertuz/segyrecover.git

First Run

To launch SEGYRecover, open Command Prompt and type:

segyrecover

On first run:

  • You'll be prompted to choose a data storage location
  • Example files will be copied to your selected location
  • The application will create the necessary folder structure

Quick Start Guide

Load an image: Click "Load Image" and select a seismic image file (supported formats: TIF, JPG, PNG)
Set parameters: Click "Parameters" to configure ROI points, acquisition parameters, and detection settings
Begin digitization: Select the region of interest by marking three corner points
Review detection: Check the baseline and timeline detection results
Complete processing: The application will extract amplitudes, process data, and create a SEGY file

VELRecover

▼

A tool for interpolating 2D seismic velocity data from sparse velocity analysis found in seismic sections.

GitHub Repository Overview

Installation

Requirements

  • Python 3.12 or higher
  • Required packages: PySide6, NumPy, pandas, appdirs, matplotlib, seisio

Using pip

pip install velrecover

From Source

pip install git+https://github.com/a-pertuz/velrecover.git

Launching VELRecover

After installation, launch the application by typing:

velrecover

Quick Start Guide

First Run Setup

When you run VELRecover for the first time:

  • You'll be prompted to choose a data storage location
  • Example files will be copied to your selected location
  • The application will create the necessary folder structure

Step-by-Step Workflow

Load Velocity Data: Click "Load Text File" and select a velocity text file with CDP, TWT, Velocity columns
Load SEGY File: Click "Load SEGY File" to provide spatial context for interpolation
Run Interpolation: Click "Interpolate" to process the data and fill gaps between sparse velocity picks
Apply Smoothing (Optional): Enter a Gaussian blur value and click "Apply Gaussian Blur"
Export Results: Save your data in text or binary format using the export buttons

InSeis: Seismic Unix Workflow Manager

▼

A graphical interface for creating and running Seismic Unix workflows on Windows through WSL.

GitHub Repository Features

Installation Prerequisites

1. Install Windows Subsystem for Linux (WSL)

Open Command Prompt as Administrator and run:

wsl --install

After installation completes, restart your computer.

2. Install Seismic Unix in WSL

Open WSL and run the installation script:

bash -c "$(wget -qO- https://gist.githubusercontent.com/a-pertuz/2b341bc8af2a37cde820d829f2789d99/raw/install_seismic_unix.sh)"

This script automates the installation of Seismic Unix release 44R28 in Ubuntu.

3. Install InSeis

Open Command Prompt and enter:

pip install inseis

Alternatively, install from GitHub:

pip install git+https://github.com/a-pertuz/inseis.git

Quick Start Guide

Launch InSeis: Open Command Prompt and type inseis
Create a workflow: The main interface displays available processes on the left and your current workflow on the right
Load data: Add a data loading process (e.g., "Load SU File") to your workflow
Add processing steps: Select processing operations from the available processes list
Run your workflow: Click "Run Workflow" to execute all steps in sequence

Important note: Seismic Unix uses SU files as its native format. If you need SEGY files for external applications, use the "Convert SU to SEGY" utility in the menu bar after processing.
