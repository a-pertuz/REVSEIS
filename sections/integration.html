Integration Guide: Creating a Complete Workflow

The three tools (SEGYRecover, VELRecover, and InSeis) can work together to form a comprehensive seismic data workflow - from legacy paper sections to modern processing and interpretation.

Step 1: Digitize Legacy Data

Tool: SEGYRecover

Convert scanned seismic sections into digital SEGY format with proper amplitude and spatial information.

Go to SEGYRecover

Step 2: Create Velocity Model

Tool: VELRecover

Use sparse velocity picks to create a complete velocity model for migration and depth conversion.

Go to VELRecover

Step 3: Process Seismic Data

Tool: InSeis

Apply advanced processing workflows using Seismic Unix to enhance and analyze the data.

Go to InSeis

Example Workflow: Legacy Section to Depth Migration

Here's a complete example showing how to take a scanned seismic section all the way through to depth migration:

1. Prepare Your Data with SEGYRecover

  1. Scan your paper seismic section at high resolution (300+ DPI)
  2. Save it as a TIFF or PNG file in the SEGYRecover IMAGES folder
  3. Create a geometry file with CDP coordinates if available
  4. Use SEGYRecover to digitize the section and export as SEGY
  5. The output will be in the SEGYRecover SEGY folder

2. Build a Velocity Model with VELRecover

  1. Create a text file with available velocity picks (CDP, TWT, Velocity)
  2. Copy your digitized SEGY file to the VELRecover SEGY folder
  3. Load both the velocity data and SEGY file in VELRecover
  4. Run interpolation to create a complete velocity field
  5. Apply smoothing as needed
  6. Export the velocity model in binary format

3. Process with InSeis

  1. Launch InSeis and create a new workflow
  2. Add "Load SEGY File" and select your digitized section
  3. Convert to SU format
  4. Add "Load Velocity" and select your velocity model
  5. Add processing steps: "Apply Bandpass Filter", "Kirchhoff Migration", etc.
  6. Configure the migration to use your velocity model
  7. Run the workflow
  8. Export the results back to SEGY format if needed

Tip: For consistent results, use the same CDP range and spatial reference across all three tools. This ensures your data and velocity model properly align during processing.

Data Transfer Between Tools

SEGYRecover → VELRecover

Copy SEGY files from SEGYRecover's SEGY folder to VELRecover's SEGY folder. The SEGY headers contain CDP information that will be used to properly place the velocity picks.

VELRecover → InSeis

Export velocity models in binary format from VELRecover, which can be directly loaded into InSeis using the "Load Velocity" process. Ensure the velocity file and SEGY file share the same CDP ranges.

InSeis → External Tools

InSeis can export processed data in SEGY format, which is compatible with most commercial interpretation software.
