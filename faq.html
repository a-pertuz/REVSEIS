General Questions

Which tool should I use for my project?

Choose based on your specific need:

  • SEGYRecover: For converting paper/raster seismic sections into digital SEGY format
  • VELRecover: For creating a complete velocity model from sparse velocity picks
  • InSeis: For processing seismic data using Seismic Unix algorithms in Windows

For complete workflows involving legacy data, you might need to use all three tools in sequence.

Do I need programming experience to use these tools?

No, all three tools provide graphical user interfaces (GUI) that don't require programming knowledge. However:

  • Basic understanding of seismic data processing concepts is helpful
  • Knowledge of Seismic Unix commands will help you get the most out of InSeis

Are these tools free to use?

Yes, all three tools are released under the GNU General Public License v3.0 (GPL-3.0), which means:

  • They are free to use, modify, and distribute
  • Modified versions must also be open-source under the same license
  • There are no warranty provisions

If using the tools for research purposes, please cite them appropriately as noted in their respective documentation.

Technical Questions

What are the system requirements?

All three tools have similar basic requirements:

  • Operating System: Windows 10 or 11 (64-bit)
  • Python: Python 3.8 or newer (automatically installed with the tools)
  • RAM: Minimum 8 GB recommended (16+ GB for large datasets)
  • Disk Space: 500 MB for installation, plus space for your data
  • Additional for InSeis: Windows Subsystem for Linux (WSL) with Ubuntu

What file formats are supported?

Each tool supports different file formats:

  • SEGYRecover:
    • Input: TIF, JPG, PNG (seismic section images)
    • Output: Standard SEG-Y format
    • Geometry: Text files (.geometry) with CDP, X, Y coordinates
  • VELRecover:
    • Input: Text files (.txt, .dat, .csv) with CDP, TWT, Velocity values
    • Seismic: SEG-Y files (.sgy, .segy) for visual context
    • Output: Text files (.txt) or binary velocity files (.bin)
  • InSeis:
    • Input: Seismic Unix files (.su) and SEG-Y files (.sgy, .segy)
    • Output: Seismic Unix files (.su), convertible to SEG-Y

Can I run these tools on Mac or Linux?

The tools are primarily designed for Windows, but:

  • SEGYRecover and VELRecover should run on macOS and Linux with Python 3.12+ and the required dependencies installed
  • InSeis is specifically designed for Windows with WSL, but you can use Seismic Unix directly on Linux without needing InSeis or you can use alternative software like OpenSeaSeis or BotoSeis

Tools Questions

What is the recommended workflow for processing legacy seismic data?

For a complete workflow with paper seismic sections:

  1. Digitize paper sections: Use SEGYRecover to convert scanned images to SEG-Y files
  2. Create velocity model: Use VELRecover to build velocity fields from sparse picks
  3. Process the data: Use InSeis to apply processing steps like filtering and migration

Each tool can also be used independently for specific tasks within your existing workflow.

How accurate is the digitization in SEGYRecover?

SEGYRecover's accuracy depends on several factors:

  • The resolution and quality of the input image (higher DPI = better results)
  • Proper parameter settings for detection
  • Correct selection of ROI points

With high-quality scans and proper settings, the digitization can achieve excellent fidelity to the original section, including preserving relative amplitude relationships.

Frequency content will be limited by the scan resolution and original section quality.

How do I ensure velocity models are properly georeferenced?

For proper georeferencing across tools:

  1. In SEGYRecover, provide a geometry file with accurate CDP to X,Y coordinate mappings
  2. When creating velocity pick files for VELRecover, use the same CDP numbers as in your seismic data

Consistency in CDP numbering across all files is key to maintaining proper spatial relationships.

How can I troubleshoot common issues?

For issues with specific tools:

  • SEGYRecover:
    • Poor timeline detection: Increase HE parameter
    • Missing baselines: Adjust TLT and BFT parameters
    • Noisy output: Adjust frequency filter parameters
  • VELRecover:
    • Artifacts in velocity model: Use Gaussian blur or try different interpolation method
    • Unrealistic velocities: Clean input data by removing outliers
    • Poor surface representation: Add more velocity picks at shallow depths
  • InSeis:
    • WSL connection issues: Reinstall WSL or check system compatibility
    • Processing errors: Check parameter values and input file format
    • Module not found: Verify Seismic Unix installation in WSL

Each tool also generates log files that can help diagnose specific issues.
